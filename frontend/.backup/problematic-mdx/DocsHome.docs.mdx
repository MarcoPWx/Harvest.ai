## Quick Start

```bash
# Start development with mocks (MSW + mock WebSocket)
npm run dev:mock
# Storybook (component docs)
npm run storybook
```

- App (mock): http://localhost:3002
- Storybook: http://localhost:6006

## Key Concepts

- Mock-first: MSW handlers + mock-socket provide an offline experience and stable demos.
- Hybrid API: `/api/generate`, `/api/generations`, `/api/generate/:id` work with mocks in dev and Supabase when configured.
- Realtime Demo UX: mock WebSocket simulates progress/stream; useful for UI iteration.
